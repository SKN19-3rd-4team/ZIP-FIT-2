# 개발 진행 순서 (최종)

> **크롤링 → 데이터 확인 → API 테스트 → 프론트엔드 개발**

---

## 📋 전체 작업 흐름

```
1. 크롤링 실행
   ↓
2. 데이터 확인 (PostgreSQL)
   ↓
3. API 테스트 (/api/docs)
   ↓
4. 응답 형식 문서화
   ↓
5. 프론트엔드 개발
```

---

## 1단계: 크롤링 실행

**목적**: PostgreSQL에 실제 공고 데이터 입력

**작업 내용**:
- [ ] `zf_crawler/크롤링.ipynb` 실행
- [ ] 크롤링 완료 확인
- [ ] 오류 발생 시 `lab/ohj/환경설정_오류해결_가이드.md` 참고

**참고 문서**:
- `lab/ohj/크롤링_실행_가이드.md`

**예상 소요 시간**: 10-30분 (데이터 양에 따라)

---

## 2단계: 데이터 확인

**목적**: PostgreSQL에 데이터가 제대로 들어갔는지 확인

**작업 내용**:
- [ ] DBeaver 또는 Docker로 데이터 확인
- [ ] `annc_all` 테이블에 최소 4-5개 이상 데이터 확인
- [ ] 공고 상태, 타입 확인

**확인 쿼리**:
```sql
SELECT COUNT(*) FROM annc_all;
SELECT annc_id, annc_title, annc_status, annc_type 
FROM annc_all 
ORDER BY created_dttm DESC 
LIMIT 10;
```

**예상 소요 시간**: 5분

---

## 3단계: API 테스트

**목적**: 실제 API 응답 형식 확인

**작업 내용**:
- [ ] Django 서버 실행 (`zf_django`)
- [ ] `/api/docs` 접속
- [ ] 각 API 엔드포인트 테스트
  - [ ] `GET /api/annc_summary`
  - [ ] `GET /api/anncs`
  - [ ] `GET /api/chathistories`
  - [ ] `POST /api/chat` (가능하다면)
- [ ] 응답 형식 확인 및 문서화

**결과 문서화**:
- `lab/ohj/api응답형식.md` 파일에 실제 응답 형식 기록

**예상 소요 시간**: 15-30분

---

## 4단계: 응답 형식 문서화

**목적**: 프론트엔드 개발을 위한 가이드 작성

**작업 내용**:
- [ ] 각 API의 실제 응답 형식 정리
- [ ] 프론트엔드 개발 가이드 작성
- [ ] 예상 데이터 구조 문서화

**예상 소요 시간**: 30분

---

## 5단계: 프론트엔드 개발

**목적**: 실제 API와 연동하여 화면 구현

**작업 순서** (점진적 개발):
1. **main.html** - `/api/annc_summary` 연동
   - 통계 숫자 표시
   - 가장 간단한 API부터 시작

2. **list.html** - `/api/anncs` 연동
   - 공고 목록 표시
   - 페이지네이션 처리

3. **chat.html** - `/api/chat` 연동
   - 채팅 메시지 전송
   - AI 응답 표시

4. **main.html** - `/api/chathistories` 연동
   - 채팅 히스토리 목록 표시

**예상 소요 시간**: 각 페이지당 1-2시간
