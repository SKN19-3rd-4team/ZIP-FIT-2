# 웹 서버 실행 가이드

> **프로젝트**: figma_django  
> **목적**: 개발 서버 실행하여 웹 페이지 확인

---

## 🚀 빠른 시작

### 1단계: 프로젝트 폴더로 이동
```bash
cd figma_django
```

### 2단계: 가상환경 활성화 (필요시)
```bash
# Conda 환경 사용 시
conda activate zipfit_env

# 또는 다른 가상환경 사용 시
# source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate     # Windows
```

### 3단계: Django 개발 서버 실행
```bash
python manage.py runserver
```

### 4단계: 브라우저에서 접속
- **랜딩 페이지**: http://127.0.0.1:8000/
- **메인 페이지**: http://127.0.0.1:8000/main/
- **사용자 정보**: http://127.0.0.1:8000/user-info/
- **AI 상담**: http://127.0.0.1:8000/chat/
- **공고 목록**: http://127.0.0.1:8000/list/

## 📋 상세 가이드

### 서버 실행 옵션

#### 기본 실행 (포트 8000)
```bash
python manage.py runserver
```
**접속 URL**: http://127.0.0.1:8000/

#### 포트 지정 실행
```bash
python manage.py runserver 8080
```
**접속 URL**: http://127.0.0.1:8080/

#### 외부 접근 허용 (같은 네트워크의 다른 기기에서 접근 가능)
```bash
python manage.py runserver 0.0.0.0:8000
```
**접속 URL**: 
- 로컬: http://127.0.0.1:8000/
- 외부: http://[내-IP주소]:8000/

## ✅ 실행 전 체크리스트

### 필수 확인 사항
- [ ] `figma_django` 폴더로 이동했는지 확인
- [ ] 가상환경이 활성화되었는지 확인 (필요시)
- [ ] 필요한 패키지가 설치되었는지 확인 (`pip install -r requirements.txt`)

### 문제 해결

#### 오류: `ModuleNotFoundError: No module named 'django'`
**해결 방법**:
```bash
# 가상환경 활성화 후
pip install django
# 또는
pip install -r requirements.txt
```

#### 오류: `Port 8000 is already in use`
**해결 방법**:
```bash
# 다른 포트 사용
python manage.py runserver 8001
```

#### 오류: `TemplateDoesNotExist`
**해결 방법**:
- `figma_django/web/templates/web/` 폴더에 모든 템플릿 파일이 있는지 확인
- `config/settings.py`의 `TEMPLATES` 설정 확인

## 📝 참고사항

### 정적 파일 (CSS, JS)
- 개발 서버에서 자동으로 서빙됨
- `STATIC_URL = '/static/'` 설정으로 `/static/` 경로에서 접근 가능
- 실제 파일 위치: `figma_django/web/static/`

### 템플릿 파일
- 위치: `figma_django/web/templates/web/`
- Django 템플릿 문법 사용 (`{% load static %}`, `{% url %}` 등)

### 데이터베이스
- **figma_django**: 현재 SQLite 사용 중 (`db.sqlite3` 파일)
- **zf_django**: PostgreSQL 사용 중 (일부 데이터만 크롤링.ipynb로 저장됨)
- 모든 공고(2024년부터 현재) PostgreSQL RDB, VectorDB로 저장할 예정 (테스트 후)
