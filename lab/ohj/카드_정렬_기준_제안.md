# 공고 카드 정렬 기준 제안

## 현재 구현된 정렬 기준 ✅

### 1차 정렬: 상태 우선순위
- **공고중** (우선순위 1) - 현재 모집 중인 공고
- **접수중** (우선순위 2) - 접수 진행 중인 공고
- **공고예정** (우선순위 3) - 곧 시작될 공고
- **마감** (우선순위 4) - 이미 마감된 공고

### 2차 정렬: 날짜순 (최신순)
- 같은 상태의 공고는 등록일(`created_dttm`) 기준으로 최신순 정렬

## 정렬 로직

```javascript
// 상태 우선순위로 먼저 정렬
// 같은 상태면 날짜순 (최신순)으로 정렬
```

## 사용자 경험 개선 효과

1. **중요한 공고 우선 표시**: 현재 모집 중인 공고가 가장 먼저 보임
2. **최신 정보 우선**: 같은 상태의 공고는 최신순으로 정렬되어 최신 정보를 쉽게 확인 가능
3. **직관적인 정렬**: 사용자가 가장 관심 있어 하는 공고가 상단에 표시됨

## 향후 개선 가능한 정렬 옵션

### 사용자 선택 가능한 정렬 옵션 (추후 구현)
1. **기본 정렬** (현재): 상태 우선순위 → 날짜순
2. **최신순**: 등록일 기준 최신순
3. **마감 임박순**: D-day 기준 (D-1, D-2, ...)
4. **기관별**: LH → SH → GH 순서
5. **유형별**: 영구임대주택 → 행복주택 → 분양주택 순서

### 정렬 UI 제안
- 드롭다운 메뉴로 정렬 옵션 선택 가능
- 현재 선택된 정렬 옵션 표시
- 정렬 아이콘 (↑↓) 표시

## 현재 구현 상태

✅ 상태 우선순위 정렬 구현 완료
✅ 날짜순 정렬 구현 완료
✅ 오늘 날짜 기준 상태 재계산 구현 완료
