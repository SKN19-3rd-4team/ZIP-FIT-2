# 포트 설정 가이드

## 현재 상황

- **Django 서버**: `8000` 포트에서 실행 중
- **브라우저 접속**: `8001` 포트로 접속 중
- **문제**: API 호출 시 404 오류 발생 (포트 불일치)

## 해결 방법

### 방법 1: 브라우저에서 8000 포트로 접속 (권장)

**가장 간단한 해결 방법입니다.**

1. Django 서버가 `8000` 포트에서 실행 중인지 확인:
   ```bash
   python manage.py runserver
   # 또는
   python manage.py runserver 8000
   ```

2. 브라우저에서 `http://127.0.0.1:8000`으로 접속

3. 이제 API 호출이 정상적으로 작동합니다.

**이유**: Django는 템플릿과 정적 파일을 모두 서빙할 수 있으므로, 프론트엔드와 백엔드를 같은 포트에서 실행하는 것이 일반적입니다.

### 방법 2: Django 서버를 8001 포트로 실행

만약 특별한 이유로 브라우저를 `8001` 포트에서 열어야 한다면:

```bash
python manage.py runserver 8001
```

그러면 브라우저에서 `http://127.0.0.1:8001`로 접속해도 정상 작동합니다.

## 권장 사항

**둘 다 8000 포트에서 실행하는 것이 좋습니다.**

이유:
1. Django는 템플릿과 정적 파일을 모두 서빙할 수 있습니다.
2. CORS 문제를 피할 수 있습니다.
3. 설정이 간단합니다.
4. 개발 환경에서 가장 일반적인 방법입니다.

## 확인 방법

브라우저 개발자 도구(F12) → Network 탭에서:
- API 호출이 `http://127.0.0.1:8000/api/...`로 가는지 확인
- 404 오류가 사라졌는지 확인

## 참고

프론트엔드와 백엔드를 다른 포트에서 실행해야 하는 경우(예: React 개발 서버와 Django API 서버 분리)에는:
- CORS 설정 필요
- `API_BASE_URL` 설정 필요
- 하지만 현재 프로젝트에서는 필요하지 않습니다.
