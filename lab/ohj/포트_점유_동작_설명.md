# 포트 점유 동작 설명

## 왜 같은 포트에서 두 서버를 실행할 수 없는가?

### 기본 원리

**같은 포트에서 두 개의 서버를 동시에 실행할 수 없습니다.**

포트는 네트워크 리소스이므로, 한 번에 하나의 프로세스만 점유할 수 있습니다.

### Windows에서의 동작

1. **첫 번째 서버가 포트를 점유**
   - `zf_django` 서버가 8000 포트를 먼저 점유
   - 포트가 이미 사용 중이면 두 번째 서버는 포트를 바인딩할 수 없음

2. **두 번째 서버의 동작**
   - 포트가 이미 사용 중이면:
     - 에러 메시지 표시: `Error: That port is already in use`
     - 또는 조용히 실패하고 다른 포트를 시도할 수 있음

3. **서버 종료 시**
   - 첫 번째 서버가 종료되면 포트가 해제됨
   - 두 번째 서버가 다시 시작되면 포트를 점유할 수 있음

## 왜 "아까는 문제가 안됬다"고 느꼈을까?

### 가능한 시나리오

1. **순차적 실행**
   - `zf_django` 서버를 먼저 실행 → 종료
   - 그 다음 `figma_django` 서버 실행
   - 이 경우 충돌이 없음

2. **하나의 서버만 실행**
   - 실제로는 한 번에 하나의 서버만 실행되고 있었을 수 있음
   - 다른 서버는 종료된 상태였을 수 있음

3. **포트 충돌이 발생했지만 인지하지 못함**
   - 두 번째 서버가 시작되지 않았지만, 첫 번째 서버가 정상 작동
   - 사용자는 첫 번째 서버만 보고 있었을 수 있음

4. **다른 포트 사용**
   - 이전에는 다른 포트를 사용했을 수 있음
   - 예: `zf_django`는 8000, `figma_django`는 8001

## 현재 상황 분석

터미널 로그를 보면:
- `zf_django`: 13:24:47 시작
- `figma_django`: 13:24:50 시작 (3초 후)

**실제로는:**
- `zf_django`가 먼저 포트 8000을 점유
- `figma_django`는 포트를 바인딩할 수 없어서 에러가 발생했을 가능성이 높음
- 또는 `zf_django`가 종료되고 `figma_django`가 포트를 점유했을 수도 있음

## 확인 방법

### 현재 실행 중인 서버 확인

PowerShell에서:
```powershell
netstat -ano | findstr :8000
```

또는:
```powershell
Get-NetTCPConnection -LocalPort 8000
```

이 명령어로 어떤 프로세스가 포트 8000을 사용하고 있는지 확인할 수 있습니다.

### 서버 시작 시 에러 확인

서버를 시작할 때 다음과 같은 에러가 나타날 수 있습니다:
```
Error: That port is already in use.
```

이 경우 포트가 이미 사용 중이라는 의미입니다.

## 해결 방법

### 방법 1: 하나의 서버만 실행 (권장)

개발 중에는 필요한 서버만 실행:
- `figma_django` 개발 시: `figma_django` 서버만 실행
- `zf_django` 개발 시: `zf_django` 서버만 실행

### 방법 2: 포트 분리

- `figma_django`: 8000 포트
- `zf_django`: 8001 포트

이렇게 포트를 분리하여 동시에 실행 가능합니다.

## 결론

**"아까는 문제가 안됬다"는 것은:**
- 실제로는 한 번에 하나의 서버만 실행되고 있었거나
- 순차적으로 실행/종료되어서 충돌이 없었을 가능성이 높습니다

**현재는:**
- 두 서버가 거의 동시에 실행을 시도하여 포트 충돌이 발생한 것으로 보입니다.

**해결책:**
- 하나의 서버만 실행하거나
- 포트를 분리하여 사용하세요.
