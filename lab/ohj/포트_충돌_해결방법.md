# 포트 충돌 해결 방법

## 문제 상황

터미널 로그를 보면:
- `zf_django` 서버가 8000 포트에서 실행 중 (13:24:47 시작)
- `figma_django` 서버도 8000 포트에서 실행 중 (13:24:50 시작)

**두 서버가 같은 포트에서 실행되어 충돌이 발생하고 있습니다!**

## 증상

404 오류 메시지에 `[name='chat_interface']`가 보이는데, 이것은 `zf_django`의 패턴입니다.
`figma_django`의 `web.urls` 패턴(`landing`, `main`, `user-info`, `chat`, `list`)이 전혀 보이지 않습니다.

이것은 **`figma_django` 서버가 아니라 `zf_django` 서버에 접속하고 있다**는 의미입니다.

## 해결 방법

### 방법 1: zf_django 서버 종료 (권장)

1. `zf_django` 서버가 실행 중인 터미널로 이동
2. `Ctrl + C` 또는 `Ctrl + Break`로 서버 종료
3. `figma_django` 서버만 실행

### 방법 2: figma_django 서버를 다른 포트로 실행

```bash
cd figma_django
python manage.py runserver 8001
```

그리고 브라우저에서 `http://127.0.0.1:8001/`로 접속

### 방법 3: zf_django 서버를 다른 포트로 실행

```bash
cd zf_django
python manage.py runserver 8001
```

그리고 `figma_django`는 8000 포트에서 실행

## 확인 방법

서버를 시작한 후 터미널에 다음 메시지가 보여야 합니다:

```
Starting development server at http://127.0.0.1:8000/
```

**중요**: 같은 포트에서 두 개의 서버를 동시에 실행할 수 없습니다!

## 권장 사항

개발 중에는:
- `figma_django` 서버만 실행 (8000 포트)
- `zf_django` 서버는 필요할 때만 실행 (다른 포트 사용)

또는:
- `figma_django`: 8000 포트
- `zf_django`: 8001 포트

이렇게 포트를 분리하여 사용하세요.
